# Betabots v0.1.0

Betabots v0.1.0 is the first stable release candidate for local open-source installation across Codex, Agent Skills, Claude Code, and Cursor.

## Highlights

- Pinned Playwright runtime dependency for the repository and copied skill installs.
- Clean install path that does not rely on sibling repositories, global npm packages, caller npm cache, or `NODE_PATH`.
- Installer now places runtime dependencies in each copied Betabots skill directory.
- Local installer rejects tracked symlinks instead of copying unsafe symlink targets into user skill/plugin directories.
- Local installer stages complete copies and dependency installation in temporary sibling directories before replacing existing skill/plugin installs, preserving prior installs on validation, copy, or npm failure.
- Explicit Chromium-only browser install command.
- Optional Linux system dependency installation for minimal images.
- Automated clean-install verifier for release checks, enforced by the `Release Gate` GitHub Actions workflow on Ubuntu and macOS with supported Node 18 and Node 20 runtimes.

## Install

```bash
git clone https://github.com/yevgeniusr/betabots.git
cd betabots
node scripts/install-deps.cjs --all
node scripts/install-browsers.cjs
tests/smoke.sh
scripts/install-local.sh all
```

`node scripts/install-browsers.cjs` downloads Chrome for Testing, Chrome Headless Shell, and FFmpeg for Chromium support for the pinned Playwright revision. It does not download Firefox or WebKit.

On minimal Linux images, run `node scripts/install-browsers.cjs --with-deps` if Playwright reports missing system libraries. That path may invoke the platform package manager.

## Verify

```bash
env -u NODE_PATH npm run verify:clean-install
```

The release verifier copies a fresh tracked tree, uses an isolated `HOME` and npm cache, installs dependencies from lockfiles with lifecycle scripts disabled, installs Chromium for Playwright 1.61.1, launches Chromium, checks standalone and plugin-embedded skill runtimes, and runs smoke without inherited `NODE_PATH`.

Use `node scripts/verify-clean-install.cjs --skip-browser-install` only for developer iteration when the machine has already run the Chromium install step for the pinned Playwright revision. With this flag, the verifier reuses the caller's Playwright browser cache, or the cache named by `PLAYWRIGHT_BROWSERS_PATH`, while keeping the temporary repository and dependency tree isolated.

GitHub Actions runs the same full verifier through the `Release Gate` workflow. The clean-install matrix is the release gate for v0.1.0 platform support and must not be replaced by source-inspection tests.
