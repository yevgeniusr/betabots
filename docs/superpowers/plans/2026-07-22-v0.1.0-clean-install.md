# Betabots v0.1.0 Clean Install Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare Betabots v0.1.0 so a clean clone and installed skill copy own their Playwright runtime dependencies and can pass smoke tests without inherited `NODE_PATH` or global packages.

**Architecture:** Add a root Node manifest and lockfile for repository verification, plus a separate pinned manifest and lockfile inside `skills/betabots` for the copied runtime. The installer will install production runtime dependencies inside each copied skill directory, while Chromium browser installation remains an explicit, separable command.

**Tech Stack:** Bash, Node.js CommonJS, npm lockfiles, Playwright Chromium, Python smoke helpers, Codex/Claude/Cursor skill directories.

## Global Constraints

- The installed runtime must not rely on sibling repositories, global npm packages, pre-existing `NODE_PATH`, or copied developer `node_modules`.
- Pin compatible dependency versions and commit lockfiles.
- Avoid downloading multiple browsers if Chromium alone is sufficient.
- Preserve Codex, Claude Code, Cursor, and Agent Skills support.
- Keep Chromium download behavior explicit and separable for CI.
- Preserve the existing complete `tests/smoke.sh` suite.
- Set release metadata to `0.1.0` consistently.
- Keep AGPL-3.0-or-later licensing metadata unchanged.
- Do not push, tag, publish packages, create a GitHub release, deploy, or read secret files.
